# Установка и обновление

## Требования

- Docker

Git желателен, но не обязателен (есть fallback через curl).

## Установка

```bash
curl -sL https://raw.githubusercontent.com/FrontendMMA/vdc-tools/main/install.sh | bash
```

Или вручную:

```bash
git clone --depth 1 https://github.com/FrontendMMA/vdc-tools.git ~/.vdc-tools/src
~/.vdc-tools/src/install.sh
```

При первой установке скрипт интерактивно спросит:
- `UPSTREAM_BASE_URL` — URL вашего LiteLLM-сервера
- `UPSTREAM_TOKEN` — API-ключ

После установки:
- Исходники сохраняются в `~/.vdc-tools/src/`
- Симлинки на команды добавляются в `~/.local/bin/`
- В `~/.zshrc` (или `~/.bashrc`) добавляется PATH

## Обновление

```bash
vdc-update
```

Команда делает `git pull` в `~/.vdc-tools/src/` и удаляет старый Docker-образ (пересоберётся при следующем запуске). Симлинки указывают на `src/bin/`, поэтому обновлённый код подхватывается автоматически.

## Структура ~/.vdc-tools/

```
~/.vdc-tools/
├── src/                    # Клон репозитория
├── .env                    # Основной конфиг (UPSTREAM_BASE_URL, UPSTREAM_TOKEN, ...)
├── projects/               # Изолированные настройки по проектам
│   └── <project-id>/
│       ├── claude-config/  # Память, сессии, settings.json Claude Code
│       └── ralphex-config/ # Конфиг ralphex
├── knowledge/              # Общая база знаний (монтируется read-only)
├── mcp/                    # Конфигурация MCP-серверов (mcp.json)
├── state/                  # LiteLLM конфиг и virtualenv
└── pnpm-store/             # Кэш pnpm (общий для всех проектов)
```

Каждый проект получает уникальный ID — SHA256 от пути к проекту (используется корень git-репозитория, если доступен).
