# vdc-claude

Запуск Claude Code интерактивно в Docker-контейнере.

## Использование

```bash
vdc-claude [--model NAME] [--yolo] [--project-dir PATH] [--print-config] [аргументы Claude...]
```

## Флаги

| Флаг | Описание |
|------|----------|
| `--model NAME` | Модель для использования (переопределяет `DEFAULT_MODEL` из `.env`) |
| `--project-dir PATH` | Директория проекта (по умолчанию — текущая или корень git-репозитория) |
| `--yolo` | Пропустить запросы на подтверждение действий (`--dangerously-skip-permissions`) |
| `--print-config` | Показать вычисленные пути и выйти |
| `-h`, `--help` | Справка |

Все остальные аргументы передаются напрямую в Claude Code. Например, `--continue` и `--resume` работают как обычно.

## Примеры

```bash
# Базовый запуск с моделью
vdc-claude --model Qwen3.5-35B-A3B

# YOLO-режим (без подтверждений)
vdc-claude --model Qwen3.5-35B-A3B --yolo

# Продолжить последнюю сессию
vdc-claude --continue

# Возобновить конкретную сессию
vdc-claude --resume

# Указать другую директорию проекта
vdc-claude --project-dir /path/to/project

# Посмотреть конфигурацию
vdc-claude --print-config
```

## Изоляция проектов

Каждый проект автоматически получает изолированное хранилище:

```
~/.vdc-tools/projects/<project-id>/claude-config/
```

Здесь хранятся:
- `settings.json` — настройки Claude Code (создаётся автоматически)
- Память и сессии Claude Code

Project ID — SHA256 от абсолютного пути к проекту. Если проект внутри git-репозитория, используется корень репозитория.

## Что происходит при запуске

1. Читает `~/.vdc-tools/.env`
2. Определяет директорию проекта (текущая → корень git)
3. Создаёт/обновляет `settings.json` с правилами безопасности
4. Запускает proxy-контейнер (если не запущен)
5. Собирает Docker-образ (если нужно)
6. Запускает Claude Code в контейнере с подключённым проектом и knowledge base
