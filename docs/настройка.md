# Настройка

## Переменные .env

Файл `~/.vdc-tools/.env` — основная конфигурация. Создаётся при установке из `.env.example`.

### Обязательные

| Переменная | Описание |
|------------|----------|
| `UPSTREAM_BASE_URL` | URL LiteLLM-сервера (например `https://litellm.example.com` или `http://host.docker.internal:4000` для локального) |
| `UPSTREAM_TOKEN` | API-ключ для LiteLLM (для локального Ollama можно использовать `ollama`) |

### Модели

| Переменная | Описание |
|------------|----------|
| `DEFAULT_MODEL` | Модель по умолчанию, если `--model` не указан |
| `SMALL_FAST_MODEL` | Модель для фоновых запросов Claude Code |

### Proxy

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `PROXY_PORT` | `8080` | Порт внутреннего proxy в Docker-сети |
| `CLAUDE_GATEWAY_TOKEN` | `dummy` | Токен между Claude Code и proxy (не upstream) |
| `REQUEST_TIMEOUT_MS` | `300000` | Таймаут запроса (5 минут) |
| `LOG_LEVEL` | `info` | Уровень логов proxy: `silent`, `info`, `debug` |
| `MAX_TOKENS_CAP` | пусто | Ограничение max_tokens в запросах (полезно при зацикливании локальных моделей, например `256`) |

### Санитизация запросов

Proxy может удалять поля из запросов перед отправкой upstream. Полезно для совместимости с LLM-бэкендами, которые не поддерживают эти поля.

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `STRIP_CONTEXT_MANAGEMENT` | `true` | Удалять поле `context_management` |
| `STRIP_THINKING` | `false` | Удалять поле `thinking` |
| `STRIP_MCP_SERVERS` | `false` | Удалять поле `mcp_servers` |
| `STRIP_CONTAINER_FIELD` | `false` | Удалять поле `container` |

### Директории (необязательные)

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `VDC_ROOT` | `~/.vdc-tools` | Корневая директория vdc-tools |
| `KNOWLEDGE_DIR` | `~/.vdc-tools/knowledge` | База знаний |
| `MCP_DIR` | `~/.vdc-tools/mcp` | MCP-конфигурация |
| `PNPM_STORE_DIR` | `~/.vdc-tools/pnpm-store` | Кэш pnpm |

## MCP-серверы

MCP-серверы подключаются через файл `~/.vdc-tools/mcp/mcp.json`. Если файл существует, он автоматически передаётся в Claude Code при запуске.

### Figma

`figma-developer-mcp` предустановлен в Docker-образе. Нужен только токен.

1. Получите Personal Access Token: `figma.com` > Settings > Personal Access Tokens
2. Создайте `~/.vdc-tools/mcp/mcp.json`:

```json
{
  "mcpServers": {
    "figma": {
      "type": "stdio",
      "command": "figma-developer-mcp",
      "args": ["--stdio"],
      "env": {
        "FIGMA_API_KEY": "figd_YOUR_TOKEN"
      }
    }
  }
}
```

### Chrome DevTools

Chromium предустановлен в Docker-образе по пути `/usr/bin/chromium`.

```json
{
  "mcpServers": {
    "chrome-devtools": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "chrome-devtools-mcp@latest"],
      "env": {
        "CHROME_PATH": "/usr/bin/chromium"
      }
    }
  }
}
```

### Несколько серверов

Объедините в одном `mcp.json`:

```json
{
  "mcpServers": {
    "figma": { ... },
    "chrome-devtools": { ... }
  }
}
```

После изменения `mcp.json` перезапустите `vdc-claude`.

## Knowledge base

Общая база знаний — директория `~/.vdc-tools/knowledge/`.

- Монтируется read-only как `/knowledge` в контейнере
- Автоматически подключается через `--add-dir /knowledge`
- Подходит для заметок, спецификаций, чек-листов, промптов, плейбуков

Положите в эту директорию любые файлы, которые должны быть доступны Claude Code во всех проектах.

## Советы по локальным моделям

- Модели семейства `qwen3` работают лучше для tool-use, чем `llama3`
- Claude Code отправляет фоновые запросы на `claude-haiku-4-5-20251001` — замапьте эту модель на быструю локальную в конфиге LiteLLM
- Если модель зацикливается в длинных размышлениях, попробуйте `MAX_TOKENS_CAP=256` в `.env`
